# テスト戦略

## 自動テスト（CI対応）

### 1. ユニットテスト

**encode モジュール** (`src/encode/tests.rs`)
- エンコードロジックの検証
- フラグメント分割
- Base64エンコーディング
- 圧縮処理

**decode モジュール** (`src/decode/tests.rs`)
- デコードロジックの検証
- フラグメント解析
- Base64デコーディング
- 解凍処理

**UI状態管理** (`src/ui/*_tests.rs`)
- EncodePanelの状態管理
- DecodePanelの状態管理
- QrWindowのページネーション
- 設定値の保持

### 2. 統合テスト

**ラウンドトリップテスト** (`tests/integration_tests.rs`)
- encode → decode の完全な往復
- 全エラー訂正レベルでの動作確認
- 大容量ファイルの処理
- 圧縮効果の検証
- フラグメント順序非依存性

### 3. CI/CD

GitHub Actionsで以下を自動実行：
```yaml
- cargo fmt -- --check     # フォーマットチェック
- cargo clippy -- -D warnings  # Linterチェック
- cargo test --verbose     # 全テスト実行
```

実行環境：
- ✅ Linux (ubuntu-latest)
- ✅ Windows (windows-latest)
- ✅ macOS (macos-latest)

## 手動テスト（GUI）

### なぜ手動テストが必要か

eframe/eguiは**即時モードGUI**のため、以下の理由で自動テストが困難：
1. ウィンドウシステムが必要（ヘッドレス環境で動かない）
2. GPUレンダリングが必要
3. イベントループが外部ライブラリに依存

### 手動テストチェックリスト

#### QRコード生成パネル
- [ ] ファイル選択ダイアログが開く
- [ ] ドラッグ&ドロップでファイルが読み込まれる
- [ ] テキスト入力モードでQR生成可能
- [ ] エラー訂正レベル選択（ComboBox）
- [ ] 圧縮オプションのトグル
- [ ] 生成ボタンクリックでQRウィンドウが開く
- [ ] エラー時に適切なメッセージが表示される

#### QRコード表示ウィンドウ
- [ ] QRコードが正しく表示される
- [ ] QRコードのドットが均一な正方形
- [ ] ページネーション（前/次ボタン）
- [ ] 行数・列数の増減（1クリックで1増減）
- [ ] 全画面表示トグル（F11キー）
- [ ] ESCキーで全画面解除/ウィンドウ閉じる
- [ ] 環境変数 FILE2QR_SCALE の反映

#### データ復元パネル
- [ ] ファイル追加/削除
- [ ] ドラッグ&ドロップでファイル追加
- [ ] 出力先ディレクトリ選択
- [ ] ハッシュ選択/選択解除
- [ ] 不足フラグメント表示
- [ ] 復元ボタンでファイル作成
- [ ] 出力フォルダを開くボタン
- [ ] エラー/成功メッセージの表示

#### マルチプラットフォーム
- [ ] Windows での日本語フォント表示
- [ ] macOS での日本語フォント表示
- [ ] Linux での日本語フォント表示
- [ ] 各OSでのファイルダイアログ動作
- [ ] DPI スケーリング（125%, 150%, 200%）

## テストの実行

### 自動テスト
```bash
# 全テスト実行
cargo test

# 特定モジュールのみ
cargo test encode::tests
cargo test decode::tests
cargo test integration

# UIロジックテスト
cargo test ui::

# 詳細出力
cargo test -- --nocapture --test-threads=1
```

### 手動テスト
```bash
# デバッグビルドで実行（高速）
cargo run

# リリースビルドで実行（実際の動作確認）
cargo run --release

# 環境変数テスト
FILE2QR_SCALE=5 cargo run
```

## テストカバレッジ

### 推奨ツール

- ロジック層（encode/decode）: **90%以上**
- UI状態管理: **70%以上**

## 既知の制限

### 自動テストできないもの
1. **QRコードの視覚的品質**
   - ドットの均一性
   - 余白の適切さ
   - アンチエイリアシング

2. **実際のQRコードリーダーでの読み取り**
   - スマートフォンカメラ
   - 専用QRコードリーダー

3. **パフォーマンス**
   - 大量QRコード生成時のメモリ使用量
   - UIの応答性

4. **実ファイルシステム操作**
   - ファイルダイアログの挙動
   - フォルダを開く機能

### 対策
- **手動テストチェックリストを定期的に実行**
- **リリース前に実機でのQRコード読み取りテスト**
- **ベンチマークテストの追加（オプション）**
