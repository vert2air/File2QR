# アプリ仕様書

## 機能概要

このツールは、大きく分けて2種類の機能から成る。

- その1: 任意のデータのQRコード生成。

  - 指定したファイルの内容、または、直接入力されたテキストの内容を、QRコードに変換し表示する。

    - QRコードに変換する際に、エラー訂正レベルを選択可能とする。

  - 圧縮オプションを有効にすると、データをxz圧縮した上でQRコードに変換する。

  - 入力されたデータの大きさが1個のQRコードでは表現し切れないケースを想定して、一般には、複数個のQRコードを生成する。GUIでは、一連のQRコードを順次表示する機能を備える。

- その2: 複数QRコードからのデータ復元。

  - 1つのデータが複数のQRコードに分割されたケースを想定して、それらのQRコードから得られたデータ列を合成し、元のデータに復元する機能。

  - 尚、QRコードの読み取り自体は、スマートフォン等を使用することを想定し、本アプリの機能には含めない。読み取られたQRコードのデータだけを受け取り、元のデータを復元する。

二つの機能を提供する実行ファイルは、1個で提供する。
理由: エンコードとデコードで別れることで、処理に齟齬が生まれないようにする為。

## 機能詳細

### QRコードに変換するデータの形式

圧縮オプションが指定された場合、
入力データがテキストデータかバイナリデータかに依らず、まず、xz圧縮を行う。
xz圧縮後のデータを入力データとして扱う。

入力データにBase64変換を行う。

入力ファイル名を特定する。
ファイルへのパスが指定された場合、パスのディレクトリ部分を取り除く。
直接入力されたテキストの場合、"(direct_text)" をファイル名とする。

入力ファイル名をパーセントエンコーディングでencodeする。

以下のデータを文字列連結したデータを作成する。hash_data と名付ける。

- パーセントエンコーディングしたファイル名
- ":"
- 圧縮オプションがあれば、"xv"。なければ、空文字列。
- ":"
- Base64化された入力データ

上記データの SHA-256 ハッシュを計算し、先頭 4 byte を取得し、16進数の文字列にする。

エラー訂正レベルに応じたQRコード1個当たりにエンコード可能なデータ長を算出する。
その値から、13を減算した値を qr_cap と命名する。
尚、13 は、以下の合計数である。
hash 8文字 + コロン 1文字 + フラグメント番号 3文字 + コロン 1文字。

上記データのbyte長に 4 を加えた値を、1コード当たりのデータ長で除算し、
QRコードが幾つ必要か算出する。QRコードが qr_num 個必要だとする。

以下のデータを文字列連結したデータを作成する。qr_data と名付ける。

- qr_num を 10進表現した文字列 (右詰め先頭に0補完で3文字)
- ":"
- hash_data

i 番目の QRコードには、以下のデータをエンコードする。
この資料中では、フラグメントと呼び、
変数 qr_frag_data[i] と名付ける。
添え字 i は、1オリジンであることに注意。(1 <= i <= qr_num)

- ハッシュ値先頭4byteの16進文字列 (8文字)
- ":"
- iを10進表現した文字列 (右詰め先頭に0補完で3文字)
- ":"
- qr_data を qr_cap ずつの byte 数に分割した i番目。

### QRコード生成機能

GUIには以下の機能を備える。

- QRコード生成機能での、入力データの指定機能

  - 変換するデータが、ファイルか、直接入力したテキストかの指定。

  - ファイルを指定するためのフィールド。

  - ファイルは、以下の何れかの方法で指定することを可能とする。

    - 対象ファイルを絶対パスで指定。

    - 対象ファイルを、アプリの起動ディレクトリからの相対パスで指定。

    - ファイルセレクタGUIによる指定。

    - ファイルをフィールドにdrag&dropすることによる指定。

  - テキストを直接入力するための、テキスト領域。

  - QRコードのエラー訂正レベルの指定。デフォルトでは L を選択する。

    - エラー訂正レベルの選択肢を、選択リストに表示する時に、
      QRコード1個で何byteをエンコード可能かの目安も表示する。

    - QRコードのエラー訂正レベルの指定では、
      画面のサイズが無駄に大きくなるのを避ける為、
      UIとしてComboBoxを使用する。

- QRコード生成機能での、QRコード表示機能。

  - QRコードを一度に何個表示するかの指定を可能にする。縦の個数、横の個数をそれぞれ指定する。

    - デフォルトでは、縦 2個、横 3個とし、つまり、6個表示する。

  - QRコードを表示するウィンドウは、ルートのウィンドウとは分離する。

    - QRコードを表示するウィンドウは、拡大縮小可能にしておき、また、縦横ともスクロール可能にする。

    - QRコードを表示するウィンドウは、ルートの画面の範囲に制約されることなく、
      大きさを変更出来ること。

    - QRコードを表示する画面に画面最大化のボタンを設けること。
      OSが提供するウィンドウシステムの画面最大化ボタンとして実現してもよい。
      画面最大化した場合の背景色は、デフォルトテーマ色を使用すること。

    - QRコードを構成するビットマップを構成するドットは、正方形でなければならない。
      このため、QRコードを拡大縮小する際にも、各ドットが正方形を保つように変換（ニアレストネイバー法）すること。

      - 拡大率 n の defaultは n = 2倍 とすること。
      - 環境変数 FILE2QR_SCALE に整数値が設定されている場合、拡大率をその値に変更すること。
      - 表示サイズは整数ピクセルに丸めることで、全QRコードで均一なドットサイズを保証すること。
      - DPI自動調整
          - OSのDPIスケーリング（125%、150%等）に自動対応する
          - 実効倍率 = ベース倍率 × DPIスケーリング（小数点は四捨五入）
          - 例：FILE2QR_SCALE=3、DPI 125% → 実効4倍（3 × 1.25 = 3.75 → 4）
          - マルチモニター環境では、ウィンドウを移動したモニターのDPIに自動的に追従する


    - QRコードの隣にQRコードと重ならないようにラベルを表示する。
      内容は、"全体の何番目のQRコード / QRコードの総数" とする。
      文字の大きさは、GUI上の他の文字と同程度にする。

    - 生成したQRコードの個数が、縦×横 の個数に収まらない場合、(縦×横)個ずつの組に分け、
      次の組の表示に移動したり、前の組の表示に戻したりを可能にする。

      - 最初の組を表示している時は、前の組を表示するボタンをdisableにする。

      - 最後の組を表示している時は、次の組を表示するボタンをdisableにする。
        それ以外は、スペースキーで、次の組を表示する。

      - 前後の組への移動は、移動用のボタンのclickに加え、左右の矢印キーでも可能にする。

    - ユーザがQRコードの生成を指示した時点では、
      QRコードは最初の組のものだけを表示するが、
      バックグラウンド処理として、全てのQRコードの生成を進めておくこと。
      これにより、以下を実現すること。

      - 表示グループが切り替わった時には、QRコードの再計算無しで、生成済みのQRコードを表示する。

    - QRコード生成を行った後も、別の入力データが与えられれば、
      新しい入力データに対し、表示の処理を進める。

### QRコードから読み取ったデータの復元機能

GUI には、以下の機能を備える。
データ復元機能においては、入力ファイルは、文字列で、
qr_frag_data[i] を完全に含む行を1行以上含むことを期待する。
qr_frag_data[i] の文字列の前後に関係ない文字列が付いている場合は、
余分な文字列を無視する。

- データ復元機能での、入力ファイルの指定機能

  - ファイルを指定するためのフィールド。

  - ファイルは、以下の何れかの方法で指定することを可能とする。

    - 対象ファイルを絶対パスで指定。

    - 対象ファイルを、アプリの起動ディレクトリからの相対パスで指定。

    - ファイルセレクタGUIによる指定。

    - ファイルをフィールドにdrag&dropすることによる指定。

  - 入力ファイルは、複数指定可能にする。

    - 入力ファイルが指定されたら、アプリケーションは直ぐにファイルの内容を
      解析し、GUIに反映する。

- データ復元機能での、出力対象データの指定機能。

  - 入力ファイル中に複数のhash値が含まれる可能性がある。
    それぞれのhash値に対応して以下を表示する。

    - 不足しているフラグメントの番号。

    - 1番目のデータが含まれる場合、ファイル名。
      1番目のデータが含まれない場合、"(unknown)" と表示。

    - 1番目のデータが含まれる場合、圧縮オプションのon/off。
      1番目のデータが含まれない場合、"--" と表示。

  - ユーザは、不足しているフラグメントが無いhash値に対し、
    デコード対象とするかどうかを指定できる。

- データ復元機能。

  - ユーザがデコード対象としたファイルを復元する。

    - 圧縮オプションがonの場合、xvファイルとして解凍し、内容のファイルを出力する。

    - ファイル名は、入力データで指定されていたファイル名を与える。

    　- ファイル名が "(direct_text)" の場合、GUI上のテキスト領域にその値を表示する。

    - 出力先のディレクトリは、以下のいずれかをユーザが選択する。

      - 入力ファイルと同じディレクトリ (default)

      - downloadディレクトリ

      - プロセスのcurrent directory

      - ユーザが指定したディレクトリ。

## 3. エラー訂正レベルと容量

容量にマージンを取ることはしない。

| レベル | 記号 | 許容誤り率 | 最大データ容量 (byte) |
|--------|------|------------|-----------------------|
| L      | L    | 7%         | 約 2,953   |
| M      | M    | 15%        | 約 2,331   |
| Q      | Q    | 25%        | 約 1,663   |
| H      | H    | 30%        | 約 1,272   |

## 4. 想定ユースケース
- 大容量ファイルを QR コードに分割して紙や画像で保存
- ネットワークを介さずにデータを物理的に転送
- テキストメッセージを QR 化して配布
